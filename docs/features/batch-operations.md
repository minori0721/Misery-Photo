# 批量操作

批量操作是高风险动作，建议先小样本验证再全量执行。

## 支持动作

- 批量删除
- 批量复制
- 批量移动
- 批量下载

批量动作可针对文件与文件夹路径混合执行。

## 推荐执行流程

1. 先在目标目录选 3-5 个样本做演练。
2. 再按目录分批执行，不建议一次性拉满 1000 条。
3. 每批执行后刷新并抽样检查结果。

## 行为规则

- 删除：文件夹会递归删除其下对象。
- 复制/移动：文件夹会按相对路径复制到目标目录。
- 移动：先复制，成功后删除源对象。

为什么这样做：

- 先复制后删除可以降低数据丢失风险。
- 发生中断时，至少能保证源数据仍在（或部分仍在）。

## 限制与边界

- 批量路径数上限为 1000。
- 单路径最大长度 1024。
- 目标目录不能在源文件夹内部（防止递归自拷贝）。

## 失败场景与处理

- 部分对象失败：记录失败路径，单独重试。
- 长时间无响应：缩小批次并降低并发。
- 目标结构异常：检查源路径是否以 `/` 结尾表示文件夹。

## 失败处理建议

- 大批次失败时优先分批重试。
- 对跨区域或高延迟桶，建议降低并发量。
- 先在小样本目录验证移动策略，再跑全量任务。

## 风险提示

- 删除动作不可恢复，建议先做一次离线备份。
- 跨桶迁移建议先用复制，再人工确认后删除源数据。
