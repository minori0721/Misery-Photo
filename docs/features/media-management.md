# 媒体管理

本页说明日常最常用的媒体浏览与播放能力，以及推荐操作方式。

## 浏览模式

系统提供两种主要浏览模式：

- 网格模式：适合快速筛选与管理。
- 漫画模式：适合顺序浏览连续内容。

建议：

- 管理场景优先网格模式（便于多选和操作按钮定位）。
- 审核或连续浏览场景优先漫画模式。

## 支持的媒体类型

- 图片：jpg、jpeg、png、webp、gif
- 视频：mp4、webm、mov、m4v、m3u8

## 视频能力

- 列表中可识别视频文件并展示播放入口。
- 支持在弹层中播放视频。
- 对 m3u8 资源可通过 hls.js 兼容播放。

设计原因：

- 将图片与视频统一在同一列表，避免“资源分库”导致的上下文切换。
- 图片 Lightbox 与视频弹层分开处理，避免误把视频当图片连续切换。

## 预览与性能

- 文件夹预览优先使用图片，避免视频首帧影响加载体验。
- 代理接口支持对大图进行压缩输出，降低首屏流量开销。

典型瓶颈：

- 单目录对象太多会导致首屏解析压力上升。
- 大图原图直出会显著增加首屏等待。

## 使用建议

- 大体积视频建议使用外部 CDN 加速。
- 对长期归档内容，建议按画集目录分层避免单目录对象过多。

## 常见误区

1. 仅按时间堆在根目录，不做目录分层。
2. 把播放卡顿完全归因于前端，而忽略存储区域与出口带宽。
3. m3u8 只上传索引文件，遗漏分片文件导致无法播放。
