# 版本迁移指南

本页用于记录重要版本变更、兼容性影响与升级步骤。

## 适用范围

- 从旧版本升级到当前 `main`。
- 从单桶配置迁移到多桶 KV 持久化配置。
- 从图片-only 流程迁移到图片+视频混合流程。

## 近期关键变更

### 1) 多桶配置迁移到服务端 KV

影响：

- 桶配置不再只依赖本地或单环境变量。
- 需要正确配置 `BUCKET_STORE_PROVIDER` 与对应 KV 变量。

升级动作：

1. 在目标环境配置 KV 连接变量。
2. 设置 `BUCKET_ENCRYPTION_KEY`。
3. 登录后台重新校验并保存桶配置。

### 2) 默认桶策略调整

影响：

- 默认桶可作为兜底可见入口。
- 默认桶通常不可删除。

升级动作：

1. 确认默认桶是否符合当前业务预期。
2. 在设置中切换激活桶并验证列表数据。

### 3) 视频能力接入

影响：

- 列表支持视频类型。
- 上传与 ZIP 导入支持视频媒体。

升级动作：

1. 验证视频上传、播放、删除流程。
2. 如使用 m3u8，确保分片文件可访问。

## 升级前检查清单

- 备份当前环境变量。
- 记录当前可用桶配置。
- 准备回滚版本号。

## 升级后回归清单

1. 登录与登出。
2. 桶列表读取、保存、切换。
3. 图片上传、视频上传、ZIP 导入。
4. 批量复制/移动/删除。
5. 下载与代理访问。

## 常见迁移问题

### 升级后桶读取失败

- 多数是 `BUCKET_ENCRYPTION_KEY` 变更导致旧密文不可读。

### 升级后某些接口 401

- 检查 `AUTH_SECRET` 是否丢失或被替换。

### 升级后部署通过但行为异常

- 优先核对 Preview/Production 环境变量是否一致。

## 回滚建议

- 使用上一个稳定提交重新部署。
- 回滚后不要立即覆盖当前 KV 数据，先确认兼容性。
- 回滚成功后再制定下一次迁移计划。
